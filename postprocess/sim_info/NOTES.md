# Generating sim_config.yml for different simulations

Most of the sim_config is constant, and is stored in this directory as
`stub.yml`.  Given tables with simulation-specific information as
described below, we can generate a sim_config file for each simulation
using a snippet of R code.

table: `exp-info.tsv`
|column | key | meaning | example |
|-------|-----|---------|---------|
|1|driving_experiment_id| scenario id| `evaluation`|
|2|driving_experiment   | scenario desc|`reanalysis simulation of the recent past` |
|3|start_year           | CORDEX starting year | `1980`|
|4|end_year             | CORDEX ending year   | `2023`|
Note: start_year and end_year may vary by simulation; driving_ entries
are controlled vocabulary from CMIP.

table: sim_info.tsv
|column | key | meaning | example |
|-------|-----|---------|---------|
|1|`id`|name used for files & directories|`mpi-hist`|
|2|`path`|sub-path within raw directory|`MPI_ESM-2-HR/hist`|
|3|`driving_source_id`|name of GCM (CV)|`MPI-ESM1-2-HR`|
|4|`driving_experiment_id`|scenario (CV)|`historical`|
|5|`driving_institution_id`|GCM insitution (CV)|`MPI-M`|
|6|`driving_variant_label`|GCM ensemble member|`r1i1p1f1`|
|7|`calendar`|GCM calendar|`proleptic_gregorian`|


## TSV file generation

The `exp-info.tsv` file is generated by hand from JSON files in the
[cordex-cmip-cv github repo](https://github.com/WCRP-CORDEX/cordex-cmip6-cv/).
(It could be auto-generated, but it's too short to bother.)

The `sim-info.tsv` file is generated by grepping information out of
attributes in files from the driving GCMs and doing a little
hand-editing.  (For example, `id` must be assigned manually.)

```
cd ~/image/collections/na-cordex-cmip6/CMIP6

set bits = (source_id experiment_id institution_id variant_label calendar)

foreach i (*/*/)
  echo -n $i
  set f = `find $i -name \*nc | head -1`
  foreach meta ($bits)
    set result = `ncdump -h $f | grep $meta | grep -Ev 'parent|sub_' | cut -f2
-d\"`
    echo -n "\t$result"
  end
  echo
end
```

## yaml file generation

Given the two TSV files and `stub.yml`, scenario-specific sim_config
files can be generated using this R code:

```
conda activate r-4.3
R
library(dplyr)

simdata <- read.table("sim-info.tsv", header=TRUE, sep='\t')
expdata <- read.table("exp-info.tsv", header=TRUE, sep='\t')

data <- merge(simdata, expdata, by='driving_experiment_id')
data <- data[,c(colnames(simdata), colnames(expdata)[-1])]
rownames(data) <- data$id
data <- data %>% mutate(across(!where(is.numeric), ~ sprintf('"%s"', .)))

for (id in rownames(data)){
    fname <- paste(id, "sim_config", "yml", sep=".")
    paste(colnames(data), data[id,], sep=": ") |> write(fname)
    system(paste("cat stub.yml >>", fname))
}
q("no")
conda deactivate
```
